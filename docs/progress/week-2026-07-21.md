## 1. CI quality gates & database testing (22 July)

**Why it matters:** Every change to the app now runs through automated checks before it can merge. This catches regressions early and gives confidence that the Supabase backend behaves correctly with real data.

### What was added

- **GitHub Actions pipeline** (`.github/workflows/ci.yml`) with two jobs:
  - **Quality** — lint, unit tests, production build
  - **Supabase local DB e2e** — starts a local Postgres via Supabase CLI, lints the schema, and runs integration tests against real rows
- **E2e test suite** (`tests/e2e/`) covering:
  - Pair lifecycle (create, save progress, claim roles, complete, comparative report)
  - Admin flows (list pairs, seed question bank, delete pair with audit)
  - Schema health (RLS enabled, required tables/functions)
- **Helper tooling** — `scripts/export-supabase-env.sh` exports local Supabase credentials for tests; new npm scripts (`test:e2e`, `test:e2e:local`, `supabase:start`, `supabase:stop`)
- **Database fix** — migration `20260721174635_grant_service_role_table_access.sql` restores `service_role` table grants needed for CI verification

### Impact for the team

- Broken builds are blocked automatically on pull requests.
- Core backend flows (pairs, participants, responses, admin) are verified on every merge.
- README updated with CI and local e2e instructions.

## 2. Accessibility & usability improvements (23 July)

**Why it matters:** The platform is now aligned with WCAG-oriented patterns so mothers, daughters, and facilitators using screen readers, keyboards, or reduced-motion settings get a better experience.

### What was added

- **Skip navigation** — keyboard users can jump past repeated chrome to main content
- **Visible focus states** — clear outlines on interactive elements
- **Screen reader support** — live announcements, proper ARIA labels, progress bars with `role="progressbar"`
- **Accessible dialogs** — replaced native `alert` / `confirm` / `prompt` with focus-managed modals (`AccessibleDialog`, `useAccessibleDialog`)
- **Chart accessibility** — radar charts include a data table fallback for non-visual users
- **Reduced motion** — respects `prefers-reduced-motion` in CSS
- **Automated a11y tests** — `vitest-axe` checks key pages (cover, role selection, test flow, dialogs)

### Pages touched

Cover, role selection, test flow, dashboards, individual/comparative reports, admin dashboard, and shared UI components.
